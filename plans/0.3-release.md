# 0.3 release plan

Status: written 2026-07-03. Milestone: [v0.3](https://github.com/elixir-waffle/waffle_gcs/issues?q=milestone%3Av0.3).
0.2.x is the released line; `main` (which becomes 0.3) is unreleased — so 0.3 is
the one chance to fix behavior and pin the consumer contract *before* anyone
depends on the broken parts of it. People aren't going to change their waffle
definitions and risk losing data; the contract we ship in 0.3 is the one we
refactor under for years.

Everything here gates cutting the release. The rewrite does **not** — 0.3 ships
on the existing Google deps.

## 0. Prerequisite: the forward-port

Land the double-filename fix (#25) and the test overhaul per
[forward-port-to-main.md](forward-port-to-main.md). Summary: hand-apply the
`put/3` / `path_for/3` / `fullname/3` delta, cherry-pick the four test commits,
don't drag the 0.2-branch token wiring across.

Why this fix is right (proven against waffle 1.1.5 source): waffle's
`put_version` resolves the filename and writes it into `file.file_name` *before*
calling the adapter (`deps/waffle/lib/waffle/actions/store.ex`), so `put/3`
must raw-join; `delete`/`url` receive the *original* name from the consumer, so
`path_for/3` must keep resolving. `main`'s PR #1 fixed `put` by gutting
`path_for` for its other two callers; this branch fixes `put` and leaves
`path_for` correct.

**Changelog note (subtle contract change):** direct callers of
`CloudStorage.put/3` (not via `definition.store`) previously had `filename/2`
resolution applied inside `put`; now they upload under the raw `file_name`.
Correct, but observable.

## 1. Lib fixes (issues #27–#34)

Ordered roughly by consumer impact:

- **#27 — ACL handling.** Waffle's default `@acl :private` is merged into
  `Object.acl` (typed `list(ObjectAccessControl.t)`) and sent as the invalid
  `"acl": "private"`, which GCS ignores — uploads work *by accident*. S3-style
  atoms (`:public_read`, what every arc_gcs/S3 migrant writes) are silently
  dropped → private objects, no error. Uniform bucket-level access buckets
  (the GCS default for new buckets) reject object ACLs outright. Fix: map atom
  ACLs → `predefinedAcl` query param, send nothing for `:private`/`nil`, keep
  list-of-maps → object `acl` field, document UBLA.
- **#34 — token story.** Decide deliberately (see issue for full detail):
  `main`'s `fetch_env!(:waffle, :token_fetcher)` breaks every 0.2 upgrader;
  `GothTokenFetcher` is conditionally compiled only when goth < 1.3 so it
  doesn't exist on modern goth; and `UrlV2` hardcodes legacy `Goth.Config`
  reads, bypassing the fetcher abstraction (root of #16, related #18).
  Direction: defaulted `get_env` lookup, a default fetcher that works on
  Goth ≥ 1.3, signing credentials threaded through the same seam.
- **#28 — asset_host plumbing.** `endpoint(opts)` is dead code — both call
  sites call `endpoint()` bare, so per-call `asset_host:` is ignored, and
  `definition.asset_host/0` is never consulted (S3/Local parity). Resolution
  order after fix: per-call opt → definition callback → app config → default.
- **#29 — signed + asset_host.** The canonical resource signed omits the
  bucket when an asset host is set, so the signature can never validate. 0.3
  decision: sign against the real GCS resource or raise; never emit a URL that
  cannot work.
- **#31 — URL encoding.** `URI.encode/1` leaves `#`/`?` unencoded → object
  names truncate at the fragment. Encode per-segment with
  `&URI.char_unreserved?/1`.
- **#30 — swallowed callback errors.** The `try/rescue UndefinedFunctionError`
  around `gcs_object_headers`/`gcs_optional_params` eats errors raised *inside*
  user callbacks → headers silently dropped. Use
  `Code.ensure_loaded?` + `function_exported?`.
- **#32 — library-owned result types.** `put/delete` currently return
  `{:ok, %GoogleApi...Object{}} | {:error, %Tesla.Env{}}`. Normalize to
  library-owned structs (with `raw:` during transition) and pin in tests — this
  is what makes the eventual client rewrite (#39) non-breaking.
- **#33 — bucket-from-scope.** The intended 0.3 feature. Also resolves the
  version-gating flip: `test_helper.exs` stops excluding
  `:bucket_with_file_and_scope` the moment the version bumps to 0.3.0, and the
  suite goes red unless this is implemented (stopgaps in
  forward-port-to-main.md §4 — decision is **option (a)**, implement it).

## 2. Test-infra hazards (issues #35, #36) — 0.3-gating

These are in-scope for 0.3, not "later": the suite is about to be the thing we
point contributors at.

- **#35 — cleanup safety.** `Cleanup.execute` wipes the *entire* bucket after
  every suite, runs unconditionally (crashes credential-less `mix test.unit`),
  and parallel CI matrix jobs sharing one bucket delete each other's in-flight
  objects. Fix: per-run object prefix + prefix-scoped cleanup; skip when
  `:integration` is excluded or creds are absent; optionally refuse
  non-test-looking bucket names.
- **#36 — CI resurrection.** The workflow triggers on `master`; the default
  branch is `main` — CI has not been running at all. Also: full `mix test` on
  PRs can't work for forks (no secrets), the matrix tops out at Elixir
  1.12/OTP 24, and `WAFFLE_BUCKET2` (now in the repo secrets) isn't passed
  through the workflow env. Split unit (every PR, no secrets) from integration
  (trusted pushes).

## 3. Test gaps (issue #37) — all of them, before release

Full checklist lives in #37. Every item lands in 0.3 — this is the released-0.2
→ unreleased-0.3 boundary, the cheapest moment to pin the contract:

1. Default content-type inference (`.png` serves as `image/png`).
2. `gcs_optional_params/2` coverage (zero today; it's the `predefinedAcl`
   escape hatch).
3. `gcs_object_headers/2` as keyword list (README documents it; only map form
   tested).
4. Deterministic signing test vector (fixed PEM + expiry → exact signature,
   offline) — enables the v4 signer work (#14) to be verified without creds.
5. Multi-version delete: assert `:thumb` is gone too, not just `:original`.
6. Special-char round trips: spaces/unicode/`#`/`%` store → GET (#31 pins).
7. Signed-URL combinations: signed+space, signed+asset_host (#29 pins).
8. Error/return contract shapes (#32 pins).
9. `assert_acls_public_reader` hardcodes `"uploads/"` — derive from the
   definition.
10. The `:bucket_with_file_and_scope` gating flip (#33).

Also from forward-port-to-main.md §4: convert future-behavior `@tag skip:`
tests into named exclude tags so they're CLI-toggleable with `--include`.
